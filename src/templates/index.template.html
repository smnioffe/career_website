Appointment No Show Model

The model was made to predict “No Shows” from clinical & claims data. “No Shows” are appointments which a patient makes, but doesn’t show up for, cancel or reschedule prior

Benefits of Predicting No Shows

Target outreach, additional appointment reminders or other services like at home visits, transportation or assistance from social workers Target the cross section of likely “No-Shows” & High-Risk patients

Past Behavior

For each appointment, the status of the previous appointment (Kept, No-show, etc.) was obtained along with number of previous appointments, and the patients trailing appointment behavior (ex. 25% cancelled, 25% no show, 50% kept)

Getting the Data

The first step was to obtain data for each appointment for a given time and relative information like time of day, when the appointment was made, etc.

Demographics

The next step was to obtain patient demographics such as age, gender as well as patient chronic conditions and past diagnoses

Distance to Appointment

Created Python script which used the Google Maps API to retrieve coordinated for each location, based on the practice name Found a lookup table to map patients ZIP codes to coordinates and used Haversine formula to find distance between the two sets of coordinates

Income

Used publicly available income information to approximate income by patient zip code

Provider Specialty

Grouped several categories and select the top reasons and specialties as features

Tuning The Model

Used Python and SQL to transform nominal features (ethnicity, gender, etc.) into binary features (ex: female: 1/0; male: 1/0). The features were normalized to have the same initial weight.

Selecting The Model

Using different hyperparameter as well as models to choose best model and parameters. Evaluated: Decision Tree vs K Nearest Neighbor vs Random Forrest vs Ridge Classifier vs Logistic Regression
